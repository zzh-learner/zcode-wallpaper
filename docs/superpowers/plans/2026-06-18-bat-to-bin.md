# .bat 集中到 `bin/` 目录 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 5 个辅助 .bat + probe.ps1 从根目录移进新建的 `bin/` 文件夹，`wallpaper.bat` 留根目录当唯一入口；同步修复所有 `%~dp0` 路径引用，使搬家后命令链不断。

**Architecture:** 纯搬家 + 路径字符串修复，不改任何功能逻辑。每个移进 `bin/` 的 .bat 开头加一行算出项目根 `set "WP_ROOT=%~dp0.."`，把原来指向 `%~dp0` 的引用改成 `%WP_ROOT%`（lib 在根下）或 `%~dp0`（probe.ps1 在 bin/ 同目录）。`wallpaper.bat` 留根，自己的 `node lib/...` 调用不变，只把 6 处 `call` 加 `bin\` 前缀。`probetest.cjs` 改一行（probe.ps1 新路径）。

**Tech Stack:** Windows batch (cmd.exe)、PowerShell (.ps1)、Node.js CommonJS (.cjs)。不加新依赖。

对应 spec：`docs/superpowers/specs/2026-06-18-bat-to-bin-design.md`

---

## 关键背景（执行者必读）

**`%~dp0` 的含义**：当前 .bat 自身所在目录（末尾带反斜杠，如 `C:\proj\bin\`）。

**为什么不能只移动不改路径**：被 `call` 的 .bat 按**自己位置**找文件。setup.bat 移到 `bin/` 后，它的 `%~dp0lib\setup.cjs` 会找 `bin/lib/setup.cjs`（不存在）→ 断。

**`set "WP_ROOT=%~dp0.."` 的原理**：`bin/` 在项目根下，根是 bin 的上一级。`%~dp0` = `C:\proj\bin\`，拼 `..` = `C:\proj\bin\..`，cmd 解析路径时这个等价于 `C:\proj`。后续 `%WP_ROOT%\lib\setup.cjs` = `C:\proj\bin\..\lib\setup.cjs` = `C:\proj\lib\setup.cjs` ✓。

**不要碰 `lib/`、`scripts/`、`test/` 里的其他文件**（除 probetest.cjs）。`lib/*.cjs` 用 `__dirname` 相对路径，只要 lib 还在根下就正常。

**所有 .bat 保持 ASCII-only**（AGENTS.md 约定）。

## Task 3: 移动并修复 `start-zcode.bat`（最复杂，涉及 probe.ps1）

`start-zcode.bat` 有两个外部引用：`probe.ps1`（现在和它同在 bin/）和 `lib/inject.cjs`（在根的 lib/）。

**Files:**
- Move + Modify: `start-zcode.bat` → `bin/start-zcode.bat`

- [ ] **Step 1: git mv**

```bash
git mv start-zcode.bat bin/start-zcode.bat
```

- [ ] **Step 2: 改 bin/start-zcode.bat 的路径**

读整个 `bin/start-zcode.bat`。有三处要改：

**(a) 项目根变量（替换原 WP_DIR 定义）**

找到（约第 18-19 行）：
```bat
set "WP_DIR=%~dp0"
set "WP_DIR=%WP_DIR:~0,-1%"
```
改成：
```bat
REM  Project root = parent of this script's dir (bin/ lives under root)
set "WP_ROOT=%~dp0.."
```
（原来的两行 strip 末尾反斜杠的逻辑删掉，因为 `%~dp0..` 不需要 strip，cmd 路径解析能处理 `bin\..`。）

**(b) probe.ps1 调用（约第 52 行）**

找到：
```bat
powershell -NoProfile -ExecutionPolicy Bypass -File "%WP_DIR%\probe.ps1" -Port %DEBUG_PORT% >nul 2>nul
```
改成（probe.ps1 现在和 .bat 同在 bin/，用 %~dp0 同目录）：
```bat
powershell -NoProfile -ExecutionPolicy Bypass -File "%~dp0probe.ps1" -Port %DEBUG_PORT% >nul 2>nul
```

**(c) inject.cjs 调用（约第 67 行）**

找到：
```bat
node "%WP_DIR%\lib\inject.cjs"
```
改成：
```bat
node "%WP_ROOT%\lib\inject.cjs"
```

**注意**：文件里可能还有其他 `%WP_DIR%` 引用（比如 echo 提示里的路径、日志路径 `zcode-launch.log`）。逐一检查：
- 如果是输出给用户看的提示文字（如 `echo [wallpaper] - Change image: edit wallpaper.css ...`），**不涉及路径定位，不用改**。
- 如果是用 `%WP_DIR%` 拼接真实文件路径（如写日志 `%WP_DIR%\zcode-launch.log`），要改成 `%WP_ROOT%\zcode-launch.log`（日志应写到项目根，不是 bin/）。

先 grep 全文确认：
```bash
grep -n "WP_DIR\|%~dp0" bin/start-zcode.bat
```
对每一处判断：是路径定位（改）还是纯文本提示（不改）。日志文件路径算路径定位，要改。

- [ ] **Step 3: 静态验证**

```bash
grep -n "WP_ROOT\|WP_DIR\|%~dp0\|probe.ps1\|inject.cjs" bin/start-zcode.bat
```
Expected:
- 有 `set "WP_ROOT=%~dp0.."`
- probe.ps1 用 `%~dp0probe.ps1`
- inject.cjs 用 `%WP_ROOT%\lib\inject.cjs`
- 日志（如有）用 `%WP_ROOT%\`
- **无任何残留 `%WP_DIR%`**

- [ ] **Step 4: 不单独提交**
